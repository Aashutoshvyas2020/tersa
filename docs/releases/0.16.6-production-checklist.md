# Tersa 0.16.6 production checklist

This release is limited to failures reproduced during real interactive use.

- [ ] Ollama model discovery refreshes every time `/model` opens and shows every installed model.
- [ ] Small local models receive a clear capability warning and a reduced-context execution policy.
- [ ] Status line reports the actual route (for example, Ollama instead of OpenAI) and has a live preview.
- [ ] Permission modes are reduced to Plan, Standard, Elevated autonomy, Perseverance, and Objective Verification.
- [ ] GSD and Superpowers cannot be enabled simultaneously; modes are simple on/off controls.
- [ ] `/copy`, `/exit`, `/color`, `/fast`, and `/effort` behave and describe themselves correctly.
- [ ] `/diff` is pinned to the active project and cannot read another session's working tree.
- [ ] Unsupported Chrome, IDE, Dream, and stale Claude-facing surfaces are removed.
- [ ] Slash-command filtering works from the first typed character.
- [ ] Tips reference only implemented features and rotate during long operations.
- [ ] A fresh session cannot inherit project-state summaries from another chat.
- [ ] Clean package install, typecheck, focused tests, full Tersa gate, PTY canary, and npm dry-run pass.
